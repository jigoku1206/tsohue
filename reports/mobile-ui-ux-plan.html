手機 UI/UX 檢查與改善計劃

這次檢查聚焦在 360px 到 430px 寬度的手機情境，特別是 dashboard、Demo dashboard、帳本/類別/個人資料管理，以及新增消費紀錄流程。

主要問題

  • 高：手機頂部工具列同時放入品牌、帳本、類別、個人資料與登出，容易壓縮到帳本名稱與按鈕觸控區。
  • 高：Demo dashboard 與正式 dashboard 的高度模型不同，手機上可能出現整頁捲動與內部列表捲動互相搶手勢。
  • 高：新增消費紀錄視窗在低高度手機上容易需要上下滑動才能完成輸入。
  • 中：帳本、類別、個人資料 dialog 使用一般桌面高度設定，小螢幕可視範圍不足時不夠穩定。
  • 中：月摘要卡與 Demo banner 在手機上佔用較多垂直空間。

改善方向

  • 讓 dashboard shell 使用固定 viewport 高度，將捲動限制在內容區。
  • 手機工具列採 icon-first 與短文字策略，保留可辨識的 aria-label 與 title。
  • 管理 dialog 使用 100dvh 安全高度，手機降低 padding 與間距。
  • 新增消費紀錄頁面採緊湊表單排列，計算機提供 compact 模式。
  • Demo banner 與月摘要卡降低手機高度，讓日曆和列表更早進入首屏。

已執行修改

區域 調整內容 預期效益
Dashboard header 縮小手機 logo、壓縮 gap、帳本按鈕限制寬度、類別與登出改為手機友善的 icon/短文字。 避免 360px 手機上工具列擠壓，提升觸控穩定性。
Demo dashboard 改為與正式版一致的固定高度 shell，內容區使用 min-h-0 與安全區 padding。 降低頁面捲動與內部列表捲動衝突。
新增消費紀錄 表單改為緊湊排列，日期/週期、分類/子分類、付款人/備註合併成手機可讀的雙欄區塊，計算機加入 compact 模式。 降低完成新增紀錄所需的垂直滑動。
管理 Dialog 帳本、類別、個人資料、Demo 個人資料 dialog 改用 max-h: calc(100dvh - 1rem) 並降低手機 padding。 小螢幕與瀏海安全區下仍可完整操作。
月摘要與 Demo banner 手機降低 padding、文字尺寸與高度，金額區避免擠壓。 首屏可顯示更多核心內容。

  • 若管理功能持續增加，可將 header 次要操作整合為一個行動版更多選單。
